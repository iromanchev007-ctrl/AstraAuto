# АстраАвтоСаратов — сайт магазина автозапчастей с онлайн-записью

Современный адаптивный лендинг магазина автозапчастей и автосервиса
«АстраАвтоСаратов» (Саратов, Октябрьская ул., 59). Это **система онлайн-записи
через сайт**: клиент оставляет марку, модель и симптомы — заявка моментально
приходит менеджеру в Telegram. Без звонков, без потерянных обращений.

Дизайн — минималистичный, бело-серо-чёрный, одинаково хорошо выглядит на ПК и
смартфонах. Плавные анимации появления блоков, hover-эффекты, SEO-разметка.

## Состав проекта

```
index.html                 — разметка страницы (Hero, О компании, Услуги и цены,
                             Заявка, Адрес, Связь, Подвал)
styles.css                 — стили, адаптив, анимации появления
script.js                  — меню, анимации, марки/модели авто, отправка формы
api/lead.js                — серверная функция отправки в Telegram (Vercel)
netlify/functions/lead.js  — то же для Netlify
netlify.toml               — конфиг Netlify (+ редирект /api/lead)
vercel.json                — конфиг Vercel (cleanUrls, заголовки безопасности)
```

## Куда вставить ключ бота и chat_id  ⬅️ главное

Токен **не вписывается в код сайта** — он живёт в переменных окружения
хостинга, поэтому не попадёт к посетителям. Нужны две переменные:

| Переменная           | Где взять                                                |
|----------------------|----------------------------------------------------------|
| `TELEGRAM_BOT_TOKEN` | у [@BotFather](https://t.me/BotFather) при создании бота |
| `TELEGRAM_CHAT_ID`   | у [@userinfobot](https://t.me/userinfobot) или [@getmyid_bot](https://t.me/getmyid_bot) |

После того как зададите переменные на хостинге, **откройте `script.js` и
поменяйте `STUB_MODE = true` на `STUB_MODE = false`** — тогда заявки начнут
реально уходить в Telegram (сейчас форма работает в демо-режиме и просто
показывает сообщение «Спасибо!»).

### Вариант А — Vercel (рекомендуется)
1. Импортируйте репозиторий на [vercel.com](https://vercel.com) (Framework Preset: **Other**).
2. **Settings → Environment Variables** → добавьте `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
3. В `script.js` поставьте `STUB_MODE = false`.
4. **Deploy** (или Redeploy, если переменные добавили позже).

### Вариант Б — Netlify
1. Импортируйте репозиторий на [netlify.com](https://netlify.com).
2. **Site settings → Environment variables** → добавьте те же две переменные.
3. В `script.js` поставьте `STUB_MODE = false`.
4. Деплой. `netlify.toml` уже настроит функцию и редирект `/api/lead`.

> Быстрый тест без переменных окружения: можно временно вписать токен и chat_id
> в блок `FALLBACK` в начале `api/lead.js` (или `netlify/functions/lead.js`).
> **Не коммитьте токен в публичный репозиторий** — он утечёт.

## Получаем chat_id за минуту
1. Создайте бота у @BotFather → получите токен.
2. Напишите своему боту любое сообщение (например, `/start`).
3. Узнайте свой ID у @userinfobot — это и есть `TELEGRAM_CHAT_ID`.
   Для группы менеджеров добавьте бота в группу и используйте ID группы
   (начинается с `-`).

## Локальный просмотр
Откройте `index.html` в браузере. Форма в демо-режиме (`STUB_MODE = true`)
сразу показывает подтверждение. Реальная отправка в Telegram заработает только
на хостинге с настроенной функцией (Vercel/Netlify) или через
`vercel dev` / `netlify dev`.
